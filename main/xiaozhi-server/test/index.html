首次配置：OpenClaw & 小智大模型 Key

第一步：配置 OpenClaw 使用的大模型以及 Key

  1. 通过 SSH 连接到云端实例。
  2. 进入 OpenClaw 项目目录并执行配置命令： 
    cd ~/open-claw/
pnpm openclaw config
  3. 根据终端提示：
    • 选择 Local 模式。
    • 选择你要用的 大模型提供商（例如 OpenAI、Moonshot、DeepSeek 等）。
    • 为对应模型填写 API Key / Base URL / 模型名称。
  4. 配置完成后，OpenClaw 会将这些信息写入它自己的配置文件，之后 xiaozhi 通过 OpenClaw 转发复杂任务就会使用这些大模型。

第二步：配置小智（xiaozhi-server）使用的大模型以及 Key

你可以在下面两种方案中选择 任意一种 完成配置：

方案一（推荐）：在网页中一键写入 DeepSeek Key

直接在下方输入 DeepSeek 的 API Key，点击按钮后会自动写入服务器上的 data/.config.yaml（仅修改 LLM.DeepSeekLLM.api_key）：

方案二：通过 SSH 编辑配置文件

  1. 通过 SSH 连接到云端实例。
  2. 打开配置文件（推荐使用覆盖配置）： 
    cd /path/to/xiaozhi-esp32-server/main/xiaozhi-server
vim data/.config.yaml
    如果 data/.config.yaml 不存在，可参考 config.yaml 中的示例拷贝一份到 data/.config.yaml 再修改。
  3. 在 LLM: 段落中选择你要用的大模型，并填写 Key，例如（以 DeepSeek 为例）： 
    LLM:
  DeepSeekLLM:
    type: openai
    model_name: deepseek-chat
    url: https://api.deepseek.com
    api_key: 你的DeepSeek API Key
  4. 如果你要用其他厂商（OpenAI、Moonshot Kimi 等），可以在 config.yaml 的 LLM 区块里找到对应示例，把 type / model_name / url / api_key 填成你自己的，然后同步到 data/.config.yaml 中。
  5. 保存文件后，重启小智服务（或下次重启时生效），小智本地对话就会使用你配置的大模型和 Key。
